# AWS CCP 강의 : 클라우드 컴퓨팅 개념 이해

**AWS Certified Cloud Practitioner (AWS CCP)** 시험을 준비하는 분들을 대상으로 **클라우드 컴퓨팅(Cloud Computing)의 개념**을 알아보겠습니다.

CCP 시험에서 가장 중요한 내용 중 하나이며, 시험 문제의 약 20~25% 정도가 클라우드 개념에서 출제됩니다.

---

# 1. 클라우드 컴퓨팅이란?

먼저 질문 하나를 해보겠습니다.

여러분이 게임을 다운로드하지 않고 인터넷으로 게임을 즐긴 적이 있나요?

또는 영화를 다운로드하지 않고 넷플릭스에서 본 적이 있나요?

클라우드 컴퓨팅도 비슷한 개념입니다.

---

## 전통적인 방식(On-Premises)

회사가 서비스를 운영하려면 직접 서버를 구매해야 했습니다.

예를 들어 쇼핑몰을 만든다고 생각해 봅시다.

회사는 다음과 같은 장비를 구매해야 합니다.

* 서버
* 스토리지
* 네트워크 장비
* 방화벽
* UPS
* 데이터센터 공간

```text
회사
 │
 ├── 서버 구매
 ├── 네트워크 구매
 ├── 스토리지 구매
 └── 직접 운영
```

문제점

* 초기 비용이 매우 큼
* 구축 기간이 오래 걸림
* 장애 발생 시 직접 처리 해야 됨
* 사용량 증가 시 추가 구매 필요함

---

## 클라우드 방식

AWS가 이미 모든 장비를 준비해 둡니다.

우리는 필요한 만큼만 사용하면 됩니다.

```text
사용자
    │
    ▼
AWS 클라우드
 ├── 서버
 ├── 스토리지
 ├── 네트워크
 └── 데이터센터
```

비유하면

> **On-Premises = 집을 직접 짓기**

> **Cloud = 호텔을 예약해서 사용하기**

---

# 2. 왜 클라우드를 사용할까?

AWS에서는 클라우드의 장점을 여러 가지로 설명합니다.

---

## ① 초기 투자 비용 감소

기존 방식

```text
서버 3대 구매

= 3천만원


사용자 수 100명
```

클라우드

```text
EC2 1대 생성

월 5만원


사용자 증가 시 추가 생성
```

필요한 만큼만 사용합니다.

---

## ② 빠른 구축

예전

```text
서버 구매

↓

배송 1주

↓

설치 2일

↓

운영 시작
```

클라우드

```text
AWS 콘솔

↓

EC2 생성

↓

5분 후 사용 가능
```

---

## ③ 탄력성(Elasticity)

사용자가 늘어나면 서버를 자동으로 늘립니다.

예시

평상시

```text
EC2 2대
```

할인 행사

```text
EC2 10대
```

행사 종료

```text
EC2 2대
```

자동으로 늘었다가 줄어드는 특징을 **탄력성**이라고 합니다.

---

## ④ 글로벌 서비스

AWS는 전 세계 여러 지역에 데이터센터를 운영합니다.

예를 들어

한국 사용자

→ 서울 리전 사용

미국 사용자

→ 버지니아 리전 사용

서비스 지연 시간을 줄일 수 있습니다.

---

# 3. 클라우드의 특징 5가지

시험에서 매우 중요합니다.

## ① On-demand Self Service

필요할 때 직접 생성

예시

EC2 생성

RDS 생성

S3 생성

별도 승인 없이 사용 가능

---

## ② Broad Network Access

인터넷만 있으면 접속 가능

예

노트북

태블릿

스마트폰

모두 사용 가능

---

## ③ Resource Pooling

AWS가 보유한 자원을 여러 고객이 공유

예

```text
AWS 데이터센터

 ├─ 고객 A
 ├─ 고객 B
 ├─ 고객 C
```

물리적으로는 공유하지만 논리적으로는 분리됩니다.

---

## ④ Rapid Elasticity

빠르게 확장 가능

1대

↓

10대

↓

100대

몇 분 안에 가능

---

## ⑤ Measured Service

사용한 만큼 비용 지불

전기요금과 비슷합니다.

```text
100시간 사용

↓

100시간 비용 지불
```

---

# 4. 클라우드 서비스 모델

시험에서 가장 많이 출제됩니다.

## ① IaaS

Infrastructure as a Service

인프라 제공

사용자가 관리

* 운영체제
* 미들웨어
* 애플리케이션

AWS 서비스

* EC2
* EBS
* VPC

비유

> 빈 건물을 임대하는 것

---

## ② PaaS

Platform as a Service

운영체제 관리 불필요

사용자는 코드만 개발

AWS 서비스

* Elastic Beanstalk
* App Runner

비유

> 주방이 있는 식당을 빌리는 것

---

## ③ SaaS

Software as a Service

바로 사용하는 서비스

예

* 이메일
* 협업도구
* 온라인 문서

대표 사례

* Gmail
* Office 365
* Slack

비유

> 음식점에서 음식을 주문하는 것

---

## 비교

| 구분            | IaaS        | PaaS                      | SaaS                   |
| --------------- | ----------- | --------------------------| ---------------------- |
| 사용 목적       | 인프라 구축 | 애플리케이션 개발         | 완성된 소프트웨어 사용 |
| 서버 관리       | 사용자      | AWS                       | AWS                    |
| OS 관리         | 사용자      | AWS                       | AWS                    |
| 애플리케이션    | 사용자      | 사용자                    | 제공됨                 |
| 개발 자유도     | 매우 높음   | 보통                      | 낮음                   |
| 대표 AWS 서비스 | EC2         | Elastic Beanstalk, Lambda | Gmail                  |

---

# 5. 클라우드 배포 모델

## ① 퍼블릭 클라우드

AWS가 제공

여러 회사가 함께 사용

가장 일반적

예

AWS

---

## ② 프라이빗 클라우드

특정 기업만 사용

은행

공공기관

---

## ③ 하이브리드 클라우드

온프레미스 + AWS

예

```text
회사 데이터센터
         │
         │ VPN
         ▼
        AWS
```

민감한 데이터는 내부 보관하고, 웹 서비스는 AWS에서 운영할 수 있습니다.


---

# 6. AWS가 제공하는 핵심 가치

AWS는 다음 6가지를 클라우드의 핵심 장점으로 설명합니다.
 
| 특징        | 설명                        |
| ----------- | --------------------------- |
| 비용 절감   | 초기 투자 감소              |
| 민첩성      | 빠른 서비스 구축            |
| 탄력성      | 자동 확장                   |
| 글로벌 확장 | 전 세계 서비스 가능         |
| 보안        | AWS의 다양한 보안 기능 활용 |
| 운영 효율성 | 서버 관리 부담 감소         |

---

# 7. CCP 시험 대비 핵심 암기 사항

| 주제          | 암기 내용                       |
| ------------- | --------------------------      |
| 클라우드 특징 | 5가지 특징 암기                 |
| 서비스 모델   | IaaS, PaaS, SaaS                |
| 배포 모델     | Public, Private, Hybrid         |
| 비용 모델     | Pay-as-you-go(사용량 기반 과금) |
| 확장 개념     | Elasticity, Scalability 구분    |

### Elasticity와 Scalability 차이

| 구분        | 설명                                      |
| ----------- | ----------------------------------------- |
| Elasticity  | 사용량 변화에 따라 자동으로 늘고 줄어듦   |
| Scalability | 시스템을 더 큰 규모로 확장할 수 있는 능력 |

---

# 정리

클라우드 컴퓨팅은 **인터넷을 통해 IT 자원(서버, 스토리지, 데이터베이스 등)을 필요할 때 필요한 만큼 사용하고, 사용한 만큼 비용을 지불하는 방식**입니다.

CCP 시험에서는 다음 네 가지를 가장 중요하게 기억하면 됩니다.

1. **클라우드의 5가지 특징**
2. **IaaS, PaaS, SaaS 서비스 모델**
3. **Public, Private, Hybrid 배포 모델**
4. **AWS의 핵심 장점(비용 절감, 탄력성, 민첩성)**
