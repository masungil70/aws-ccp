# AWS CCP - 클라우드 배포 모델(Deployment Model)

AWS CCP 시험에서는 "**클라우드 배포 모델(Cloud Deployment Model)**"을 이해하는 것이 매우 중요합니다.

오늘은 다음 4가지를 쉽게 이해해 보겠습니다.

1. 퍼블릭 클라우드 (Public Cloud)
2. 프라이빗 클라우드 (Private Cloud)
3. 하이브리드 클라우드 (Hybrid Cloud)
4. 멀티 클라우드 (Multi Cloud)

---

# 1. 퍼블릭 클라우드 (Public Cloud)

먼저 클라우드를 이해해야 합니다.

예전에는 회사가 서비스를 운영하려면 직접 서버를 구매해야 했습니다.

예를 들어,

> 쇼핑몰을 만들고 싶다.

그러면 회사는 다음과 같은 장비를 구매해야 했습니다.

* 서버 컴퓨터
* 저장장치(Storage)
* 네트워크 장비
* 보안 장비
* UPS(무정전 전원장치)
* 데이터센터 공간

예를 들어 서버 한 대가 500만원이라면,

```
서버 3대 = 1500만원
스토리지 = 500만원
네트워크 장비 = 300만원

총 약 2,300만원 이상
```

그리고 서버가 고장 나면 직접 수리해야 합니다.

클라우드는 이러한 장비를 직접 구매하지 않고,

> 인터넷을 통해 필요한 만큼 빌려 사용하는 방식

입니다.

---

# 2. 퍼블릭 클라우드란?

## 정의

퍼블릭 클라우드는

> 클라우드 서비스 제공업체가 소유한 IT 자원을 인터넷을 통해 여러 고객에게 제공하는 방식 입니다.

쉽게 말하면,

> 거대한 컴퓨터를 여러 사람이 함께 사용하는 서비스

라고 생각하면 됩니다.

대표적인 서비스 제공 업체

* Amazon Web Services
* Microsoft
* Google

# 4. AWS에서 퍼블릭 클라우드는 어떻게 제공될까?

AWS는 전 세계 곳곳에 대규모 데이터센터를 운영합니다.

사용자는 인터넷만 연결되어 있으면 언제 어디서나 서버를 생성할 수 있습니다.

예시

학생이 웹 서비스를 만들고 싶다고 가정해봅시다.

### 전통 방식 (직접운영 방식)

```text
서버 구매
      ↓
OS 설치
      ↓
네트워크 구성
      ↓
보안 설정
      ↓
서비스 운영
```

몇 주~몇 달 소요됩니다

---

### AWS 퍼블릭 클라우드

```text
AWS 콘솔 접속
        ↓
EC2 생성
        ↓
웹 서버 설치
        ↓
서비스 시작
```

약 5~10분 소요

# 5. 퍼블릭 클라우드의 특징

## ① 여러 고객이 함께 사용

AWS의 **물리적 서버**는 여러 기업과 사용자가 함께 사용합니다.

하지만 **논리적**으로는 완전히 분리되어 있습니다.

예를 들어

```text
AWS 데이터센터


┌──────────────────────┐
│ Physical Server      │
│                      │
│ 회사A VM             │
│ 회사B VM             │
│ 대학생 프로젝트 VM   │
│ 스타트업 VM          │
└──────────────────────┘
```

사용자들은 서로의 데이터를 볼 수 없습니다.

---

## ② 사용한 만큼 비용 지불

필요한 만큼만 사용합니다.

예시

```text
서버 2시간 사용

스토리지 100GB 사용

데이터 전송 50GB


→ 사용량만 과금
```

이를

> **Pay-As-You-Go (종량제 과금)**

라고 합니다.

---

## ③ 빠른 확장성

시험 기간에 사용자가 증가했다고 가정해봅시다.

평상시

```text
현재 서버 2대 운영 중 입니다
```

시험기간

```text
서버 20대 증설이 필요합니다
```

시험 종료

```text
다시 서버 2대로 운영됩니다
증설된 18대 필요 없습니다
```

몇 분 안에 조정할 수 있습니다.

이를

> **Elasticity (탄력성)**

라고 합니다.

---

## ④ 전 세계 어디서나 사용 가능

AWS는 여러 지역(Region)에 데이터센터를 운영합니다.

예시

* Seoul Region
* Tokyo Region
* Virginia Region

서비스 이용자는 가장 가까운 지역을 선택하여 서비스를 운영할 수 있습니다.

# 2. 프라이빗 클라우드 (Private Cloud)란?

---

## 1. 프라이빗 클라우드란?

**프라이빗 클라우드(Private Cloud)**는 **하나의 조직(회사, 학교, 기관)만을 위해 전용으로 사용하는 클라우드 환경**입니다.

쉽게 말하면,

> **"우리 회사만 사용하는 전용 클라우드 데이터센터"**

라고 생각하면 됩니다.

일반적인 클라우드는 여러 회사가 함께 사용하는 아파트라면,

프라이빗 클라우드는 **한 회사가 단독으로 사용하는 단독주택**과 비슷합니다.

---

## 2. 프라이빗 클라우드 이해하기

### 비유로 설명하기

| 구분                | 비유              |
| ------------------- | ----------------- |
| 퍼블릭 클라우드     | 아파트            |
| 프라이빗 클라우드   | 단독주택          |
| 하이브리드 클라우드 | 단독주택 + 아파트 |

### 아파트(퍼블릭 클라우드)

* 여러 사람이 함께 거주
* 관리비 절약
* 필요할 때 쉽게 입주 가능

### 단독주택(프라이빗 클라우드)

* 한 가족만 사용
* 보안성 높음
* 관리 비용 높음
* 원하는 대로 설계 가능

---

## 3. 프라이빗 클라우드의 특징

### ① 특정 조직만 사용

다른 기업과 자원을 공유하지 않습니다.

예시

```
A회사 전용 서버
A회사 전용 스토리지
A회사 전용 네트워크
```

오직 A회사 직원들만 사용합니다.

---

### ② 높은 보안성

금융권이나 정부 기관에서 선호합니다.

이유는

* 고객 정보 보호
* 중요 데이터 외부 유출 방지
* 내부 정책 적용 가능

예시

- 은행 고객 정보

- 병원 환자 기록

- 국가기관 행정 데이터

---

### ③ 자원 통제 가능

회사가 직접 관리할 수 있습니다.

CPU 사용량

메모리 사용량

네트워크 정책

방화벽 설정

접근 권한

모두 직접 제어 가능합니다.

---

### ④ 높은 구축 비용

단점도 있습니다.

서버 구매

스토리지 구매

네트워크 장비 구매

데이터센터 운영

비용이 많이 듭니다.

---

## 4. 프라이빗 클라우드 구조

```text
회사 내부 사용자
        │
        ▼
 ┌───────────────┐
 │ 인증 시스템   │
 └───────────────┘
        │
        ▼
 ┌───────────────┐
 │ 가상화 플랫폼 │
 │ VMware        │
 │ OpenStack     │
 └───────────────┘
        │
 ┌──────┼──────┐
 ▼      ▼      ▼
VM1    VM2    VM3

DB     WEB    APP
```

가상화 기술을 이용하여 여러 개의 서버를 효율적으로 운영합니다.

---

## 5. 프라이빗 클라우드 장점

| 장점        | 설명                         |
| ----------- | ---------------------------- |
| 높은 보안성 | 외부 사용자와 자원 공유 없음 |
| 성능 보장   | 자원을 독점 사용             |
| 맞춤형 환경 | 원하는 시스템 구성 가능      |
| 규제 대응   | 금융, 의료, 공공기관 적합    |
| 데이터 통제 | 기업 내부 정책 적용 가능     |

---

## 6. 프라이빗 클라우드 단점

| 단점              | 설명                       |
| ----------------- | -------------------------- |
| 초기 투자 비용 큼 | 서버 및 네트워크 구축 필요 |
| 운영 인력 필요    | 전문 관리자 필요           |
| 확장 속도 느림    | 장비 구매 시간이 필요      |
| 유지보수 부담     | 장애 대응을 직접 수행      |

---

## 7. 프라이빗 클라우드를 사용하는 사례

### 금융기관

은행 고객 정보 관리

예시:

* 인터넷 뱅킹 시스템
* 계좌 정보 관리

### 병원

* 전자 의료 기록(EMR)
* 환자 데이터 보관

### 정부기관

* 주민등록 시스템
* 세금 관리 시스템

### 대기업

* 기업 내부 ERP
* 인사 시스템
* 연구개발 데이터

## 8. AWS에서의 프라이빗 클라우드

AWS는 기본적으로 **퍼블릭 클라우드 서비스**이지만, 기업이 전용 환경을 구성할 수 있는 서비스를 제공합니다.

대표적인 예시는 다음과 같습니다.

* **전용 네트워크 환경(VPC)**
* **전용 물리 서버(Dedicated Host)**
* **전용 물리 인스턴스(Dedicated Instance)**

이를 활용하면 다른 고객과 물리적 자원을 공유하지 않는 환경을 구축할 수 있습니다.

---

## 9. AWS CCP 시험 핵심 정리

| 구분        | 프라이빗 클라우드    |
| ----------- | -------------------- |
| 사용자      | 단일 조직            |
| 보안        | 매우 높음            |
| 비용        | 높음                 |
| 확장성      | 상대적으로 낮음      |
| 관리        | 조직이 직접 수행     |
| 대표 사용처 | 금융, 의료, 공공기관 |

# 3. 하이브리드 클라우드(Hybrid Cloud)란?

---

## 1. 하이브리드 클라우드의 정의

**하이브리드 클라우드(Hybrid Cloud)**는

> **기업이 사용하는 자체 데이터센터(On-Premises)와 퍼블릭 클라우드를 함께 사용하는 방식**입니다.

쉽게 말하면,

> **"회사 컴퓨터실 + AWS를 동시에 사용하는 것"**

이라고 생각하면 됩니다.

---

## 2. 왜 하이브리드 클라우드를 사용할까?

모든 기업이 처음부터 AWS로 모든 시스템을 옮길 수는 없습니다.

예를 들면,

* 오래된 ERP 시스템
* 생산 관리 시스템
* 금융 거래 시스템
* 병원 환자 정보 시스템

등은 쉽게 이전하기 어렵습니다.

그래서 기업은 다음과 같이 운영합니다.

| 구분             | 운영 위치       |
| ---------------- | --------------- |
| 기존 업무 시스템 | 회사 데이터센터 |
| 웹 서비스        | AWS             |
| AI 분석          | AWS             |
| 백업 시스템      | AWS             |

즉,

> **기존 시스템은 그대로 유지하면서 필요한 부분만 AWS를 사용하는 방식**

입니다.

## 3. 대학생이 이해하기 쉬운 예시

### 학교 서버 예시

A대학교가 있다고 가정해 보겠습니다.

현재 학교에는 서버실이 있습니다.

운영 중인 시스템

* 학사관리 시스템
* 학생 정보 시스템
* 도서관 시스템

하지만 새로운 서비스를 만들려고 합니다.

* AI 챗봇
* 모바일 앱
* 빅데이터 분석

이를 위해 AWS를 사용합니다.

구성은 다음과 같습니다.

```text
                학생

                  │
                  │

     ┌────────────────────┐
     │     AWS Cloud      │
     │                    │
     │ EC2                │
     │ Lambda             │
     │ S3                 │
     │ AI 서비스          │
     └────────────────────┘
               │
           VPN / 전용회선
               │
     ┌────────────────────┐
     │ 학교 서버실        │
     │                    │
     │ 학사관리           │
     │ 도서관             │
     │ 학생DB             │
     └────────────────────┘
```

이것이 바로 **하이브리드 클라우드**입니다.

### ① 기존 시스템 활용 가능

새로 구축하지 않아도 됩니다.

기존 서버를 그대로 사용할 수 있습니다.

예시

```text
ERP → 회사 데이터센터

AI 분석 → AWS
```

### ② 단계적 클라우드 전환 가능

한 번에 모든 시스템을 이전하지 않아도 됩니다.


---

# 4. 멀티 클라우드 (Multi Cloud) 란?

---

## 1. 멀티 클라우드의 개념

**멀티 클라우드(Multi-Cloud)** 는 **두 개 이상의 서로 다른 클라우드 서비스 제공업체(CSP)를 함께 사용하는 방식**입니다.

쉽게 말하면,

> "하나의 회사가 AWS만 사용하는 것이 아니라 AWS, Azure, GCP 등을 동시에 사용하는 것"

입니다.

예를 들어,

| 서비스           | 사용하는 클라우드 |
| -----------------| ----------------- |
| 웹 서버          | AWS               |
| AI 모델 학습     | GCP               |
| 사내 업무 시스템 | Azure             |

이렇게 여러 클라우드를 조합하여 사용하는 것이 멀티 클라우드입니다.

---

## 2. 대학생 비유로 이해하기

대학교 생활을 예로 들어보겠습니다.

### 하나의 클라우드 사용

> 학교 도서관만 이용한다.

장점

* 이용 방법이 익숙하다.
* 관리가 쉽다.

단점

* 도서관이 공사하면 공부할 곳이 없다.

---

### 멀티 클라우드 사용

> 학교 도서관, 시립도서관, 카페를 모두 이용한다.

장점

* 도서관이 문을 닫아도 다른 곳에서 공부 가능
* 장소별 장점을 활용 가능
* 사람이 많으면 다른 곳 이용 가능

---

## 3. 멀티 클라우드 구조 예시

```text
                    사용자
                       │
        ┌──────────────┼──────────────┐
        │                             │
        ▼                             ▼
      AWS                           Azure
 (웹 서비스)                      (AD 인증)
        │
        │
        ▼
      GCP
(AI/BigQuery)
```

예시

AWS

* 웹 서버 운영
* 데이터 저장

Azure

* 기업 인증 서비스
* Microsoft 제품 연동

GCP

* 머신러닝
* 데이터 분석

---

## 4. 왜 멀티 클라우드를 사용할까?

### ① 각 클라우드의 강점을 활용

| 클라우드  | 강점                     |
| --------- | ------------------------ |
| AWS       | 서비스 종류가 매우 많음  |
| Azure     | Microsoft 제품 연동 우수 |
| GCP       | AI, 빅데이터 분석 강점   |

예시

AWS

* EC2
* S3

Azure

* Active Directory

GCP

* Vertex AI

---

### ② 특정 업체 종속(Vendor Lock-in) 방지

Vendor Lock-in이란,

> 특정 클라우드 서비스에 지나치게 의존하는 상황

입니다.

예를 들면,

회사 시스템 전체를 AWS만 사용하면

AWS 가격 인상 시 영향을 크게 받을 수 있습니다.

멀티 클라우드를 사용하면

> "필요하면 다른 클라우드로 일부 서비스를 이전"

할 수 있습니다.

---

### ③ 장애 대비

만약 AWS 특정 리전에 장애가 발생하면

```text
AWS 장애 발생

↓

일부 서비스는 Azure에서 계속 운영
```

서비스 중단 위험을 줄일 수 있습니다.

---

### ④ 비용 최적화

클라우드마다 가격 정책이 다릅니다.

예시

| 서비스         | 저렴한 곳 |
| -------------- | --------- |
| 객체 저장      | AWS       |
| AI GPU         | GCP       |
| Windows 서버   | Azure     |

가장 경제적인 서비스를 선택할 수 있습니다.

---

## 5. 멀티 클라우드의 단점

### ① 관리가 어렵다

AWS 콘솔

Azure Portal

GCP Console

세 가지를 모두 배워야 합니다.

---

### ② 보안 관리 복잡

각 클라우드마다

* IAM 정책
* 네트워크 설정
* 접근 권한 관리

방법이 다릅니다.

---

### ③ 운영 비용 증가

관리 인력이 더 필요합니다.

예시

AWS 전문가 1명

Azure 전문가 1명

GCP 전문가 1명

필요할 수도 있습니다.

---

## 6. AWS CCP 시험 관점에서 알아둘 내용

AWS CCP 시험에서는 멀티 클라우드를 다음과 같이 이해하면 됩니다.

> **멀티 클라우드(Multi-Cloud)는 둘 이상의 서로 다른 클라우드 제공업체의 서비스를 함께 사용하는 전략이다.**

핵심 목적은 다음과 같습니다.

* 특정 업체 의존성 감소
* 서비스 가용성 향상
* 비용 최적화
* 각 클라우드의 장점 활용

---

## 7. 시험 예상 문제

### 문제

멀티 클라우드 전략의 가장 큰 장점은 무엇인가?

A. 단일 클라우드보다 관리가 쉽다.

B. 하나의 공급자에 대한 의존도를 줄일 수 있다.

C. 보안 설정이 단순해진다.

D. 항상 비용이 절감된다.

### 정답

✅ **B. 하나의 공급자에 대한 의존도를 줄일 수 있다.**

## AWS CCP 핵심 정리

| 구분      | 설명                                         |
| --------- | -------------------------------------------- |
| 정의      | 두 개 이상의 클라우드 제공업체를 함께 사용   |
| 대표 사례 | AWS + Azure + GCP                            |
| 장점      | 벤더 종속성 감소, 가용성 향상, 비용 최적화   |
| 단점      | 관리 복잡성 증가, 보안 관리 어려움           |
| 시험 핵심 | **Vendor Lock-in 방지**가 가장 중요한 키워드 |

AWS CCP 시험에서는 **멀티 클라우드(Multi-Cloud)** 와 **하이브리드 클라우드(Hybrid Cloud)** 의 차이를 자주 묻기 때문에, 두 개념을 함께 비교해서 이해하는 것이 중요합니다.
