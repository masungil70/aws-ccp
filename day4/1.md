# AWS CCP 비용 관리 및 지원 서비스

## 과금 모델(On-Demand, Reserved, Spot, Savings Plans)

AWS Certified Cloud Practitioner(CCP) 시험에서는 **비용 최적화(Cost Optimization)** 가 약 24% 정도 출제됩니다.

이번 장에서는 AWS에서 가장 많이 출제되는 과금 모델인 다음 5가지를 학습합니다.

* 과금 모델(Pricing Model)
* On-Demand
* Reserved
* Spot
* Savings Plans

---

# 1. AWS 과금 모델(Pricing Model)

## 과금 모델이란?

AWS에서는 서버를 구매하지 않습니다.

필요한 만큼 사용하고 사용한 만큼만 비용을 지불(Pay As You Go)하는 방식입니다.

즉,

> "자동차를 사는 것이 아니라 렌트하는 것"

과 비슷합니다.

예를 들어

* 하루만 사용할 수도 있고
* 1시간만 사용할 수도 있고
* 1년 동안 사용할 수도 있습니다.

사용 목적에 따라 가장 적절한 과금 모델을 선택하면 됩니다.

---

## AWS 과금 모델 종류

| 모델          | 특징                           |
| ------------- | ------------------------------ |
| On-Demand     | 필요할 때 바로 사용            |
| Reserved      | 장기간 사용 시 할인            |
| Spot          | 남는 자원을 매우 저렴하게 사용 |
| Savings Plans | 일정 금액 사용을 약속하면 할인 |

AWS 시험에서는

> "어떤 상황에서 어떤 과금 모델을 사용하는가?"

를 매우 많이 묻습니다.

---

# 2. On-Demand

## 개념

On-Demand는

> **필요할 때 생성하고 필요 없으면 삭제하는 가장 기본적인 과금 방식**입니다.

예약도 없고 계약도 없습니다.

언제든 시작하고

언제든 종료할 수 있습니다.

---

## 특징

* 선불 없음
* 약정 없음
* 시간(또는 초) 단위 과금
* 자유롭게 생성/삭제 가능
* 가장 비싼 요금

---

## 사용 예

대학교 실습

학생들이

EC2를 생성했다가

수업이 끝나면 삭제

→ On-Demand

---

개발 서버

필요할 때만 실행

→ On-Demand

---

## 장점

* 가장 간단
* 유연함
* 언제든 종료 가능

---

## 단점

장기간 사용하면

비용이 많이 발생합니다.

---

# 예시

```
EC2

월요일 생성
화요일 삭제

사용한 시간만 과금
```

---

# 3. Reserved Instance (RI)

## 개념

Reserved는

> "1년 또는 3년 사용할 것을 미리 약속"

하면

AWS가 큰 할인을 제공합니다.

---

## 특징

* 1년 또는 3년 약정
* 최대 약 72% 할인(조건에 따라 다름)
* 장기간 운영에 적합

---

## 사용 예

회사 홈페이지

365일 운영

절대 종료하지 않음

→ Reserved

---

데이터베이스 서버

24시간 운영

→ Reserved

---

## 장점

* 매우 큰 할인
* 예측 가능한 비용

---

## 단점

* 약정 필요
* 사용량이 줄어도 약정은 유지

---

# Reserved가 적합한 서비스

* EC2
* RDS
* ElastiCache
* OpenSearch 등 일부 서비스

---

# 4. Spot Instance

## 개념

Spot은

AWS 데이터센터에서

남아있는 EC2 자원을

매우 저렴하게 빌려주는 서비스입니다.

---

## 가장 중요한 특징

AWS가 자원이 필요하면

언제든 회수할 수 있습니다.

이를

Interrupt(중단)

라고 합니다.

즉

```
사용자
    ↓
Spot Instance 실행

AWS 자원 부족

↓

2분 전에 알림

↓

인스턴스 종료
```

---

## 특징

* 최대 약 90% 할인(시장 상황에 따라 달라짐)
* 언제든 종료될 수 있음
* 안정성보다 비용 절감

---

## 사용 예

AI 학습

영상 렌더링

배치 작업

빅데이터 분석

CI/CD 테스트

---

## 적합하지 않은 경우

은행 시스템

쇼핑몰

ERP

데이터베이스

실시간 서비스

---

## 장점

엄청난 비용 절감

---

## 단점

언제든 종료 가능

---

# 5. Savings Plans

## 개념

Savings Plans는

> 일정 사용량(예: 시간당 $10)을 1년 또는 3년 동안 사용하겠다고 약속하면 할인받는 모델입니다.

Reserved Instance보다 훨씬 유연합니다.

---

## 특징

* 1년 또는 3년 약정
* 최대 약 72% 할인(조건에 따라 다름)
* EC2 유형 변경 가능
* 리전 변경이 가능한 유형도 있음(플랜 종류에 따라 다름)
* 자동 적용

---

## Savings Plans 종류

### ① Compute Savings Plans

가장 많이 사용

적용 대상

* EC2
* AWS Fargate
* AWS Lambda

가장 유연합니다.

---

### ② EC2 Instance Savings Plans

특정 EC2 패밀리에 집중

가장 높은 할인

---

## 장점

Reserved보다 유연

자동 할인

관리 쉬움

---

## 단점

약정 필요

---

# 최종 비교

| 비교 항목       | On-Demand         | Reserved Instance | Spot Instance  | Savings Plans    |
| --------------- | ----------------- | ----------------- | -------------- | ---------------- |
| 약정            | 없음              | 1년/3년           | 없음           | 1년/3년          |
| 할인율          | 없음              | 최대 약 72%       | 최대 약 90%    | 최대 약 72%      |
| 유연성          | 매우 높음         | 낮음              | 높음(중단 위험)| 높음             |
| 인스턴스 중단   | 없음              | 없음              | 있음           | 없음             |
| 비용            | 가장 높음         | 낮음              | 가장 낮음      | 낮음             |
| 대표 용도       | 개발·테스트, 실습 | 운영 서버, DB     | 배치, AI 학습, 렌더링 | 장기 운영 + 유연한 워크로드 |
| CCP 시험 빈도   | ★★★★★             | ★★★★★             | ★★★★★          | ★★★★★            |

---

# 암기법(CCP 시험 대비)

| 상황                                   | 선택해야 할 과금 모델                        |
| -------------------------------------- | -------------------------------------------- |
| 잠깐 사용한다                          | **On-Demand**                                |
| 항상 사용하는 서버                     | **Reserved Instance** 또는 **Savings Plans** |
| 가장 저렴하게 사용하고 중단되어도 된다 | **Spot Instance**                            |
| 장기 사용하면서도 인스턴스 변경이 잦다 | **Savings Plans**                            |
| 개발/실습 환경                         | **On-Demand**                                |
| AI 학습, 대규모 배치 작업              | **Spot Instance**                            |

---

# CCP 시험 핵심 포인트

1. **On-Demand**: 약정 없이 필요할 때 사용하는 기본 과금 모델.
2. **Reserved Instance**: 특정 리소스를 1년 또는 3년 사용할 때 높은 할인 혜택을 제공.
3. **Spot Instance**: 남는 EC2 용량을 매우 저렴하게 사용할 수 있지만, AWS의 필요에 따라 중단될 수 있음.
4. **Savings Plans**: 일정 사용량을 약정하여 EC2뿐만 아니라 AWS Fargate와 AWS Lambda까지 유연하게 할인받을 수 있는 최신 비용 절감 모델.
5. 시험에서는 **"워크로드의 특성(일시적, 장기 운영, 중단 허용 여부, 유연성 요구)"에 따라 어떤 과금 모델을 선택해야 하는지**를 묻는 문제가 매우 자주 출제됩니다.

> **시험 팁**
> CCP에서는 "가장 저렴한 비용"만 묻는 것이 아니라, **비용과 운영 요구사항을 함께 고려한 최적의 선택**을 묻는 문제가 많습니다. 예를 들어, 장기간 운영이지만 인스턴스 타입 변경 가능성이 있다면 **Savings Plans**가, 중단이 허용되는 AI 학습이나 배치 작업이라면 **Spot Instance**가 가장 적합한 선택입니다.
